# TSK-0123: Skip PlaceBlock at Bot's Current Position (No Reference Block)

**Priority:** P0 — Critical
**Status:** Backlog
**Sprint:** 50 (Wave A)
**Theme:** Build Placement Correctness
**Source:** Live production log 2026-06-25

## Description

The agent repeatedly tries to place a block at its own standing position
(-236,66,173) where there's no solid reference block. The PlaceBlock
operation fails with "no solid reference block" and the replan keeps
producing the same action because world state doesn't change.

Log evidence:
```
[22:53:56] [action] MoveTo OK (0ms) pos=(-236,66,173)
[22:53:56] [action] PlaceBlock OK (0ms) pos=(-236,66,173)
[22:53:56] [action] GetStatus OK (0ms) pos=(-236,66,173)
[22:53:57] Game error after cycle (failures=1/3): place:Cannot place oak_log
  at (-236,66,173) - no solid reference block (last face error:
  Event blockUpdate:(-236, 66, 173) did not fire within timeout of 5000ms)
  block=oak_log at (-236,66,173)
```

## Evidence

- The bot's position equals the PlaceBlock target position in all axes.
- `PlaceBlock` in Mineflayer requires at least one adjacent solid block as a
  reference face — you can't place a block in the space the bot occupies.
- The build plan doesn't filter out positions that match the bot's current
  location, causing repeated failures.
- Related to TSK-0122 (blocks occupied by vegetation) — both are about
  preconditions for successful block placement.

## Requirements

1. In `HtnTaskLibrary.DecomposeBuild`, before emitting PlaceBlock actions,
   filter out any block position that matches the bot's current position
   from WorldState.
2. These positions will be filled on the next replan cycle once the bot has
   moved to place adjacent blocks.
3. Add logging when a position is skipped due to bot occupancy.

## Acceptance Criteria

- [ ] PlaceBlock is never emitted for the bot's current position
- [ ] Skipped positions are logged at Debug level
- [ ] Adjacent blocks placed in the same cycle provide a reference for
      skipped positions on the next replan
- [ ] `dotnet build` passes with zero warnings
- [ ] All existing tests pass
