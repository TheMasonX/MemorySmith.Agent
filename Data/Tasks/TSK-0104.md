# TSK-0104: ReplanResult Type + ReplanGoalContext Fix

**Priority:** P1 — High
**Status:** Backlog
**Sprint:** 46 (Wave B)
**Theme:** Observability First
**Council:** audit-validation-sprint46-plan-council-2026-06-24

## Description

Two related problems in the replanning path:

1. **ReplanAsync returns null for multiple failure modes:** `null` can mean "planner crashed" (catch-all), "PlanAsync returned null", or "unknown goal type". Callers cannot distinguish.
2. **ReplanAsync ignores originalGoal:** The `IGoal? originalGoal` parameter is accepted but NEVER used. Replanning creates a bare `SimpleGoal` from the plan name string, losing all typed goal data (blueprint ID, origin coordinates, item specs, etc.).

## Evidence

- **Validated claims:** 4 (planner swallows errors), 8 (replan loses goal context)
- **Sources:** `HtnPlanner.cs:169-196`, `IPlanner.cs` interface

## Requirements

1. Introduce `ReplanResult<IPlan>` sealed record:
   - `bool Success`
   - `IPlan? Plan`
   - `string? FailureReason`
   - `string? ExceptionType`
   - `ReplanFailureCategory` enum: `Timeout | NotFound | Exception | Stalled | NoAction`

2. Change `ReplanAsync` return type from `Task<IPlan?>` to `Task<ReplanResult<IPlan>>`

3. Fix `ReplanAsync` to actually USE the `originalGoal` parameter:
   - When `originalGoal` is a `BuildGoal`, preserve the blueprint ID and origin
   - When `originalGoal` is a `CraftItemGoal`, preserve the item spec
   - Fall back to plan-name reconstruction only when `originalGoal` is null

4. Update all callers to check `result.Success` instead of `result.Plan != null`

5. Add unit tests: BuildGoal → replan → typed data preserved; CraftItemGoal → replan → item spec preserved

## Acceptance Criteria

- [ ] `ReplanResult<IPlan>` sealed record with `Success`, `Plan`, `FailureReason`, `ExceptionType`, `FailureCategory`
- [ ] `ReplanAsync` returns `ReplanResult<IPlan>` — callers check `result.Success`
- [ ] `originalGoal` parameter is consumed — typed goal data survives replanning
- [ ] New tests: BuildGoal replan preserves blueprint and origin; CraftItemGoal replan preserves item spec; null originalGoal falls back to string reconstruction
- [ ] All existing tests pass
- [ ] `dotnet build` passes with zero warnings
