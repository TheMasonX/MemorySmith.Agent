# TSK-0101: Fix All catch→null Blocks with Structured Logging

**Priority:** P0 — Critical
**Status:** Backlog
**Sprint:** 46 (Wave A)
**Theme:** Observability First
**Council:** audit-validation-sprint46-plan-council-2026-06-24

## Description

Seven locations across the codebase use bare `catch { return null; }` blocks that silently swallow exceptions. This makes LLM integration bugs, planner failures, and event deserialization errors invisible until a user reports "the bot ignored me."

Each location needs a `LogWarning` with exception type and context before returning null. No behavioral change — just structured diagnostics.

## Locations

| # | File | Line | Context |
|---|---|---|---|
| 1 | `Agent.Planning/Llm/AnthropicProvider.cs` | 62-64 | LLM API call — 3 consecutive catches, final one is bare catch-all |
| 2 | `Agent.Planning/Llm/GeminiProvider.cs` | 59-61 | Same pattern as Anthropic |
| 3 | `Agent.Planning/Llm/OpenAICompatibleProvider.cs` | 73-75 | Same pattern as Anthropic |
| 4 | `Agent.Planning/LlmChatInterpreter.cs` | 330 | `ParseDecision` — swallows ALL JSON parse errors |
| 5 | `Agent.Planning/LlmChatInterpreter.cs` | 403 | `TryParseTruncatedJson` — can't tell truncated from malformed |
| 6 | `Agent.Planning/HtnPlanner.cs` | 193-196 | `ReplanAsync` — ALL planner exceptions swallowed |
| 7 | `Agent.World.Minecraft/WebSocketBridge.cs` | 378 | Event deserialization — malformed events dropped silently |

## Requirements

Each `catch { return null; }` block must be replaced with:

```csharp
catch (Exception ex)
{
    logger?.LogWarning(ex, "{Context}: {Message}", contextName, ex.Message);
    return null;
}
```

Where `{Context}` uniquely identifies the location (e.g., "AnthropicProvider.CompleteAsync", "HtnPlanner.ReplanAsync").

No behavioral change — the return value remains `null`. The fix is purely diagnostic.
