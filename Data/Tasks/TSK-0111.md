## Description

Three bare `catch` sites remain in the codebase after Wave A's TSK-0101 sweep. These are in `RestMemoryGateway.cs` and `WebSocketBridge.cs` (×2). Each silently swallows exceptions without logging. Add `LogWarning` before silent fallback to make failures visible.

- **Source:** `Agent.Memory/RestMemoryGateway.cs:82`, `Agent.World.Minecraft/WebSocketBridge.cs:96`, `Agent.World.Minecraft/WebSocketBridge.cs:480`
- **Validated claims:** Council finding M-1 (LLM provider catches)
- **Council:** Retrieval Specialist flagged RestMemoryGateway 404 swallowing as highest risk

## Requirements

1. `RestMemoryGateway.cs` line ~82: Add `LogWarning` before silent fallback
2. `WebSocketBridge.cs` line ~96: Add `LogWarning` before silent fallback
3. `WebSocketBridge.cs` line ~480: Add `LogWarning` before silent fallback

Each fix follows the same pattern as TSK-0101:
```csharp
catch (Exception ex)
{
    logger?.LogWarning(ex, "{Context}: {Message}", contextName, ex.Message);
    return null; // or equivalent fallback
}
```
