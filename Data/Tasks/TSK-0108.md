# TSK-0108: Clean Up Redundant State Duplication

**Priority:** P3 — Low
**Status:** Backlog
**Sprint:** 47+
**Theme:** Observability First
**Council:** audit-validation-sprint46-plan-council-2026-06-24

## Description

Three instances of redundant state tracking exist in AgentBackgroundService:

1. **`_currentGoal` field + `"currentGoal"` world-state fact:** Both track the active goal. The fact-tracked `"currentGoal"` is meant for dashboard visibility, not business logic. Remove the redundant field-access path.
2. **Dual replan-throttle mechanisms:** `ReplanGovernor` (fingerprint-based) + `AgentBackgroundService._lastReplanAt` (time-interval-based) both throttle replanning independently. Consolidate into one.
3. **Dual ConcurrentDictionary for PlaceBlock:** `_correlatedActions` + `_placeBlockContexts` both track PlaceBlock lifecycle. Sprint 44 P1-2 acknowledged this by adding cleanup for orphaned `_placeBlockContexts`. The proper fix is to merge.

## Evidence

- **Validated claim:** M-4 (state duplication)
- **Sources:** `AgentBackgroundService.cs:153,158,168,186-190`

## Requirements

1. Make `_currentGoal` the sole authoritative source — remove `"currentGoal"` fact access from business logic paths (keep fact write for dashboard visibility)
2. Consolidate dual replan-throttle mechanisms into one — prefer `ReplanGovernor` over `_lastReplanAt`
3. Merge `_placeBlockContexts` into `_correlatedActions` — remove one `ConcurrentDictionary`

## Acceptance Criteria

- [ ] Single state tracking for each concern (goal, replan throttle, place block)
- [ ] All existing tests pass
- [ ] `dotnet build` passes with zero warnings
- [ ] No behavioral changes
