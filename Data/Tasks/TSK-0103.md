# TSK-0103: BuildOrigin Value Object Consolidation

**Priority:** P1 — High
**Status:** Backlog
**Sprint:** 46 (Wave B)
**Theme:** Observability First
**Council:** audit-validation-sprint46-plan-council-2026-06-24

## Description

Consolidate all BuildOrigin resolution into a single `BuildOrigin` sealed record. This eliminates three overlapping problems:

1. **Partial coordinates treated as explicit:** `HasExplicitOrigin => OriginX.HasValue || OriginY.HasValue || OriginZ.HasValue` (OR, not AND)
2. **Sentinel `(0,0,0)` overload:** means "auto-detect", "fact not found", "fact unparseable", and "actual value is 0" simultaneously
3. **Three different origin-resolution paths:** GoalFactory, BuildGoalDecomposer, and HtnPlanner each have their own logic

## Evidence

- **Validated claims:** 1 (partial coordinates), 2 (zero sentinel), 3 (planner swallows), 14 (policy duplicated across 3 layers)
- **Sources:** `BuildGoal.cs:64-68`, `BuildFactKeys.cs:17`, `BuildGoalDecomposer.cs:30-36,84-101`, `HtnPlanner.cs:155-166`, `GoalFactory.cs:117-123`
- **Existing tasks:** TSK-0098 (ReadOriginFact consolidation) — absorbed, TSK-0095 (all-axes origin) — absorbed

## Requirements

1. Create `BuildOrigin` sealed record in `Agent.Core/Models/`:
   - `OriginSource` enum: `Explicit`, `StoredFact`, `AutoScan`, `PlayerPosition`
   - `int? X, Y, Z` — present only when resolved
   - `string? ResolutionNote` — optional diagnostic metadata
   - `bool IsExplicit => Source == OriginSource.Explicit`
   - `bool IsResolved => X.HasValue && Y.HasValue && Z.HasValue`
2. Replace `int? OriginX/Y/Z` in `BuildGoal` with `BuildOrigin? Origin`
3. Replace `HasExplicitOrigin` property semantics: require ALL 3 axes for explicit
4. Replace both `ReadOriginFact` helpers (BuildGoalDecomposer + HtnPlanner) with a single shared resolver
5. Update `GoalFactory.CreateAsync` to validate origin completeness before creating BuildGoal
6. Add unit tests: partial coords rejected, all-null accepted (auto-detect), explicit coords forwarded

## Acceptance Criteria
